How to use hexo write blog

Hexo 基本命令

Hexo+github建站基本逻辑

所有博客都在本地 ==/hexo/source/_posts/== 内完成，之后利用 ==hexo g== 重新生成，==hexo d== 上传到github仓库，相应对网站内容进行更新。

注意 每次上传都会删掉GitHub仓库里面前一版本的内容，注意备份。想到的解决办法为将gihub仓库clone到本地，这样每次修改后本地可利用git及时抓取最新的版本进行备份。

hexo d 发布到github会存在十几分钟的延迟，并且每次主题有大的变化时需要先hexo clean一下。

Hexo常用命令

hexo new “postName” # 新建文章

hexo new page “pageName” # 新建页面

hexo generate == hexo g # 生成静态页面至public目录

hexo server == hexo s # 开启预览访问端口（默认端口4000，”ctrl + c”关闭server）

hexo deploy == hexo d # 部署到github

hexo help #查看帮助

hexo version # 查看Hexo的版本

组合命令

hexo s -g #生成并本地预览

hexo d -g #生成并上传
